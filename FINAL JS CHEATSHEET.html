üèõÔ∏è Object-Oriented Programming (OOP)

What is OOP?

Object-Oriented Programming is a way of designing programs by bundling related data and behavior into "objects." Instead of having separate variables and functions, you create self-contained "blueprints" (classes) that can be used to build consistent, predictable objects. It helps in writing more modular, reusable, and maintainable code.

The Core Idea: A Real-World Analogy

Imagine you have a blueprint for a car. The blueprint itself is not a car; it's a detailed plan (`class`). Using this blueprint, you can build many actual cars (`objects` or `instances`). Each car will have its own properties (color, mileage) but will share the same fundamental behaviors (start engine, accelerate). OOP applies this same logic to code.

1. The Blueprint: `class` and `constructor`

A `class` is the template. The `constructor` is a special method inside the class that runs when you create a new object. Its job is to set up the initial properties of the object.

// The blueprint for all animals
class Animal {
    // The constructor initializes new objects created from this class
    constructor(name, age) {
        // 'this' refers to the specific instance of the object being created
        this.name = name;
        this.age = age;
    }

    // A method: a function that belongs to the class
    eat() {
        return `${this.name} is eating.`;
    }

    makeSound() {
        return `A generic animal sound.`;
    }
}

// Create an object (instance) from the class
const genericAnimal = new Animal('Creature', 5);
console.log(genericAnimal.name); // Output: Creature
console.log(genericAnimal.eat()); // Output: Creature is eating.

2. Building on the Blueprint: Inheritance with `extends` and `super`

Inheritance allows a new class (child) to adopt the properties and methods of an existing class (parent). The `extends` keyword establishes this relationship. The `super()` keyword is used within the child's constructor to call the parent's constructor, ensuring the parent part of the object is set up correctly.

// The 'Dog' class INHERITS from the 'Animal' class
class Dog extends Animal {
    constructor(name, age, breed) {
        // 'super()' calls the constructor of the parent class (Animal)
        // This is REQUIRED before you can use 'this' in a child constructor
        super(name, age);
        
        // Now we add properties specific to the Dog class
        this.breed = breed;
    }

    // A method specific to Dog
    bark() {
        return 'Woof! Woof!';
    }
}

const myDog = new Dog('Rex', 3, 'German Shepherd');

// myDog can use methods from BOTH Animal and Dog classes
console.log(myDog.eat()); // Inherited from Animal -> 'Rex is eating.'
console.log(myDog.bark()); // Defined in Dog -> 'Woof! Woof!'
console.log(myDog.breed); // 'German Shepherd'

3. Different Forms, Same Action: Polymorphism

Polymorphism means "many forms." In OOP, it's the ability for a child class to provide its own unique implementation of a method that it inherited from a parent class. The method name is the same, but the behavior is different.

class Cat extends Animal {
    constructor(name, age, color) {
        super(name, age);
        this.color = color;
    }
    
    // This is POLYMORPHISM. We are overriding the parent's makeSound method.
    makeSound() {
        return 'Meow!';
    }
}

// Let's also override it for the Dog class for a better example
class Dog extends Animal {
    constructor(name, age, breed) {
        super(name, age);
        this.breed = breed;
    }
    
    makeSound() {
        return 'Woof!';
    }
}

const theDog = new Dog('Buddy', 4, 'Golden Retriever');
const theCat = new Cat('Whiskers', 2, 'Tabby');

// We can treat different objects in a similar way
const pets = [theDog, theCat];
pets.forEach(pet => {
    // We call the same method, but get a different result for each object!
    console.log(`${pet.name} says: ${pet.makeSound()}`);
});
// Output:
// Buddy says: Woof!
// Whiskers says: Meow!

4. Hiding Complexity: Encapsulation with Getters, Setters, and Private Fields

Encapsulation is about bundling data and methods together and controlling access to that data. Modern JavaScript allows for truly private fields using a `#` prefix. We can then expose controlled access using `getters` (to read data) and `setters` (to validate and write data).

class Player {
    // #score is a private field. It CANNOT be accessed from outside the class.
    #score = 0;
    #maxHealth = 100;
    _health = 100; // By convention, _ indicates a "protected" property

    constructor(name) {
        this.name = name;
    }

    // A 'getter' allows you to read the private data. It looks like a property.
    get currentScore() {
        return `Score for ${this.name}: ${this.#score}`;
    }

    // A 'setter' allows you to modify the private data, often with validation.
    set health(newHealth) {
        if (newHealth < 0) {
            console.error("Health cannot be negative!");
        } else if (newHealth > this.#maxHealth) {
            this._health = this.#maxHealth;
        } else {
            this._health = newHealth;
        }
    }
    
    addPoints(points) {
        this.#score += points; // We can modify it from inside the class
    }
}

const player1 = new Player('Knight');
player1.addPoints(50);

// console.log(player1.#score); // ‚ùå SyntaxError: Private field must be declared in an enclosing class

// Use the getter to read the score
console.log(player1.currentScore); // 'Score for Knight: 50'

// Use the setter to change health
player1.health = 80;
console.log(player1._health); // 80

A `static` method or property belongs to the `class` itself, not to an `instance` of the class. You call it directly on the class name. They are often used for utility functions that relate to the class.

Array.prototype.reduce() Method

What is `reduce()`?

The reduce() method executes a "reducer" callback function on each element of the array, resulting in a single output value. It's used to condense an array into a single value (e.g., sum, product, average, or even a new object).

Syntax: array.reduce(callback(accumulator, currentValue, currentIndex, array), initialValue)

accumulator

The value that accumulates the callback's return value. It's the "running total" or "result so far."

  • If `initialValue` is provided, `accumulator` starts as `initialValue`.
  • If `initialValue` is *not* provided, `accumulator` starts as the first element of the array.
  • In subsequent calls, `accumulator` is the value returned by the *previous* callback invocation.

currentValue

The current element being processed in the array during the iteration.

  • If `initialValue` is provided, `currentValue` starts as the first element.
  • If `initialValue` is *not* provided, `currentValue` starts as the second element (as the first element became the initial `accumulator`).

`initialValue` (Optional)

The value to use as the first argument (`accumulator`) to the first call of the `callback`. This is highly recommended for clarity and to avoid errors with empty arrays.

  • If provided, the callback runs for every element.
  • If *not* provided, the first element is used as `accumulator`, and iteration starts from the second element. **Caution: Throws `TypeError` if array is empty.**

Example: Summing All Numbers

const numbersToSum = [1, 2, 3, 4, 5];

const totalSum = numbersToSum.reduce((acc, curr) => {
    console.log(`Accumulator: ${acc}, Current Value: ${curr}`);
    return acc + curr; // The returned value becomes 'acc' for the next iteration
}, 0); // Initial value for accumulator is 0

console.log(`Final Sum: ${totalSum}`);
/*
Execution Trace:
1. Initial: acc = 0, curr = 1. Returns 1.
2. Next:    acc = 1, curr = 2. Returns 3.
3. Next:    acc = 3, curr = 3. Returns 6.
4. Next:    acc = 6, curr = 4. Returns 10.
5. Next:    acc = 10, curr = 5. Returns 15.
Final result: 15
*/

Example: Calculating Average (Complex Accumulator)

const grades = [85, 90, 78, 92, 88];

const averageGrade = grades.reduce((acc, grade, index, array) => {
    // acc starts as { sum: 0, count: 0 }
    acc.sum += grade; // Add current grade to the running sum
    acc.count++;      // Increment the count

    // If this is the last element, calculate and return the final average
    if (index === array.length - 1) {
        return acc.sum / acc.count;
    }

    // Otherwise, return the updated accumulator object for the next iteration
    return acc;
}, { sum: 0, count: 0 }); // Initial value is an object to track sum and count

console.log(`Average Grade: ${averageGrade}`); // Output: Average Grade: 86.6

Calling Functions and Storing Returned Values

How to Pass Values Between Functions

Functions can produce values using the return keyword. To use that value in another part of your code or in another function, you must "capture" it by assigning the function call to a variable.

// Function that returns a value
function getUserName() {
    let name = prompt('Please enter your name:');
    return name; // Returns the name string
}

// Function that needs a value from another function
function greetUser() {
    // 1. Call getUserName() and store its returned value in 'userName'
    const userName = getUserName();

    // 2. Use the captured value
    if (userName) {
        alert(`Welcome, ${userName}!`);
    } else {
        alert("Welcome, Guest!");
    }
}

‚ö†Ô∏è What Happens If You Don't Capture the Return Value?

If a function returns a value, but you don't assign its call to a variable, that value is simply discarded. The function still runs, but its output is ignored.

    return 123;

What are Regular Expressions?

Regular Expressions are powerful patterns used to match character combinations in strings. They are used for searching, replacing, and validating text.

// 2. RegExp Constructor (useful for dynamic patterns)

RegEx Flags

g (Global)

Finds all matches, not just the first one.

const text = "apple banana apple orange";
const regex = /apple/g;

i (Case-Insensitive)

Performs case-insensitive matching.

const text = "Hello hello HELLO";
const regex = /hello/gi;

m (Multiline)

Treats beginning (^) and end ($) anchors as working on each line, not just the whole string.

const text = "Line 1\nLine 2\nLine 3";
const regex = /^Line/gm;
text.match(regex); // ["Line", "Line", "Line"]

  • \d: Any digit (0-9)
  • \D: Any non-digit
  • \w: Any word character (a-z, A-Z, 0-9, _)
  • \W: Any non-word character
  • \s: Any whitespace character
  • \S: Any non-whitespace character

Quantifiers

  • *: 0 or more times
  • +: 1 or more times
  • ?: 0 or 1 time (optional)
  • {n}: Exactly n times
  • {n,}: n or more times
  • {n,m}: Between n and m times

Anchors & Boundaries

  • ^: Start of string/line
  • $: End of string/line
  • \b: Word boundary
  • \B: Non-word boundary

Groups & Alternation

  • ( ): Grouping and capturing
  • |: OR operator (e.g., cat|dog)
  • [ ]: Character set (e.g., [aeiou])
  • [^ ]: Negated character set (e.g., [^0-9])

Promises (ES6)

Solving Callback Hell

Promises represent the eventual completion (or failure) of an asynchronous operation and its resulting value. They make asynchronous code easier to read and manage.

// Creating a Promise
function fetchDataPromise() {
    return new Promise((resolve, reject) => {
        setTimeout(() => {
            const success = true;
            if (success) {
                resolve("Data from server (promise)");
            } else {
                reject("Error fetching data (promise)");
            }
        }, 1500);
    });
}

// Consuming a Promise
fetchDataPromise()
    .then(data => {
        console.log(data); // Handles successful resolution
        return "Processed: " + data;
    })
    .then(processedData => {
        console.log(processedData); // Chaining .then() calls
    })
    .catch(error => {
        console.error(error); // Handles rejections (errors)
    })
    .finally(() => {
        console.log("Promise finished (always runs)"); // Always runs, regardless of success/failure
    });

// Promise.all() - run multiple promises concurrently
const p1 = new Promise(res => setTimeout(() => res(1), 500));
const p2 = new Promise(res => setTimeout(() => res(2), 200));
const p3 = new Promise(res => setTimeout(() => res(3), 800));

Promise.all([p1, p2, p3])
    .then(results => {
        console.log("All promises resolved:", results); // [1, 2, 3] (after 800ms)
    })
    .catch(error => console.error(error));

The Event Loop

How JavaScript Handles Concurrency

The Event Loop is a fundamental concept that explains how JavaScript executes code, handles asynchronous operations, and manages the call stack and callback queue.

// Synchronous code runs first (Call Stack)

// Asynchronous Web API (e.g., setTimeout)

    console.log("3. setTimeout callback (Task Queue)");

    console.log("2. Promise callback (Microtask Queue)", res);

// 1. Start
// 4. End
// 2. Promise callback (Microtask Queue) Promise resolved
// 3. setTimeout callback (Task Queue)

// Explanation:
// 1. Sync code runs.
// 2. setTimeout is a Web API; its callback goes to the Task Queue.
// 3. Promise.resolve().then() callback goes to the Microtask Queue.
// 4. Sync code finishes.
// 5. Event Loop checks Microtask Queue (higher priority) -> runs Promise callback.
// 6. Event Loop checks Task Queue -> runs setTimeout callback.

‚ö†Ô∏è DOM Manipulation Don'ts / Best Practices

  • **Avoid `innerHTML` with untrusted input**: It can lead to Cross-Site Scripting (XSS) vulnerabilities. Use `textContent` if you only need to insert plain text.
  • **Minimize direct DOM access in loops**: Repeatedly querying or modifying the DOM inside a loop can be slow. Build HTML strings or document fragments, then insert once.

  • **Use Event Delegation**: For lists or elements that are dynamically added, attach a single event listener to a common parent instead of many individual listeners.
  • **Consider CSS for animations**: For complex animations, CSS transitions and animations often perform better than JavaScript-driven animations.

üíæ Local Storage (Web Storage API)

What is localStorage?

localStorage is a browser feature that allows you to store key-value pairs in a web browser with no expiration time. Data persists even after the browser is closed and reopened. It is part of the Web Storage API and is useful for saving user preferences, themes, or small amounts of data on the client side.

All localStorage Methods

setItem(key, value)

Stores a value under the specified key. If the key already exists, its value is overwritten. Only strings are stored; use JSON.stringify() for objects/arrays.

// Store a string
localStorage.setItem('theme', 'dark');

// Store an array
let colors = ['red', 'green'];
localStorage.setItem('colors', JSON.stringify(colors));

// Overwrite existing key
localStorage.setItem('theme', 'light');

getItem(key)

Retrieves the value for the given key. Returns null if the key does not exist. Use JSON.parse() to retrieve objects/arrays.

// Get a string
let theme = localStorage.getItem('theme'); // 'light'

// Get and parse an array
let colors = JSON.parse(localStorage.getItem('colors'));

// Handle missing key
let missing = localStorage.getItem('notThere'); // null
if (missing === null) {
  console.log('Key not found!');
}

removeItem(key)

Removes the key and its value from storage. Does nothing if the key does not exist.

// Remove a key
localStorage.removeItem('theme');

// Try to remove a non-existent key (no error)
localStorage.removeItem('notThere');

clear()

Removes all keys and values from storage for the current origin. Use with caution!

// Clear all localStorage data
localStorage.clear();

// After clear, all keys are gone
localStorage.getItem('theme'); // null

key(index)

Returns the key at the specified index (0-based) in storage. Useful for iterating all keys.

// Get the first key
let firstKey = localStorage.key(0);

// Loop through all keys
for (let i = 0; i < localStorage.length; i++) {
  let key = localStorage.key(i);
  let value = localStorage.getItem(key);
  console.log(`${key}: ${value}`);
}

length (property)

Property that returns the number of stored items in localStorage for the current origin.

// Get the number of items in localStorage
let count = localStorage.length;
console.log(`There are ${count} items in localStorage`);
  • Iterating all keys/values: Use localStorage.length and localStorage.key(index) in a loop to access all items.
  • Best practice: Always check for null when using getItem to avoid errors with missing keys.

Basic Usage

// Store a value
localStorage.setItem('key', 'value');

// Retrieve a value
let value = localStorage.getItem('key');

// Remove a value
localStorage.removeItem('key');

// Clear all storage
localStorage.clear();

Storing Arrays and Objects

localStorage only stores strings. To store arrays or objects, use JSON.stringify() to save and JSON.parse() to retrieve.

Example 1: Store a String

localStorage.setItem('username', 'alice');
let user = localStorage.getItem('username'); // 'alice'

Example 2: Store an Array

let colors = ['red', 'green', 'blue'];
localStorage.setItem('colors', JSON.stringify(colors));
// Retrieve and parse
let savedColors = JSON.parse(localStorage.getItem('colors'));
// ['red', 'green', 'blue']

Example 3: Store an Object

let userObj = { name: 'Bob', age: 30 };
localStorage.setItem('userObj', JSON.stringify(userObj));
// Retrieve and parse
let savedUser = JSON.parse(localStorage.getItem('userObj'));
// { name: 'Bob', age: 30 }

Use Cases

  • Saving user preferences (theme, language, layout)
  • Persisting form data between sessions
  • Storing shopping cart items in e-commerce sites
  • Remembering login state (with caution!)

Best Practices

  • Always use JSON.stringify and JSON.parse for non-string data.
  • Do not store sensitive information (passwords, tokens) in localStorage.
  • Be mindful of storage limits (usually ~5MB per origin).
  • Handle null returns from getItem (missing keys).
  • Consider using sessionStorage for data that should not persist after the tab is closed.

üêõ JavaScript Errors: Understanding and Solving Them

Errors are Your Friends!

Errors are not failures; they are signposts telling you exactly where your code needs attention. Learning to read and understand them is the most critical skill for a developer. This section will guide you through the most common types of errors, why they happen, and how to fix them.

A Step-by-Step Guide to Debugging

  1. Read the Error Message Carefully: The console provides the error type (e.g., `TypeError`), a description, and the file name with the exact line number where the error occurred. This is your starting point.
  2. Go to the Line Number: Examine the line of code mentioned in the error message. Often, the problem is on that line or the line directly above it.
  3. Use `console.log()`: Before the error line, print the variables involved to the console. Are they the values you expect? Are they the correct data type? `console.log('My variable:', myVariable);`
  4. Check Your Assumptions: Did you assume a variable would be a number when it's actually a string? Did you assume an object would have a certain property that it doesn't?
  5. Isolate the Problem: Comment out code blocks until the error disappears. Then, reintroduce code line by line until the error reappears. The last line you added is the culprit.
  6. Use the Debugger: For complex issues, use the browser's built-in debugger. Set a "breakpoint" on the line before the error, and you can step through your code one line at a time, inspecting the state of all variables as you go.

Handling Errors Gracefully: `try...catch`

Sometimes you anticipate that code might fail (e.g., parsing invalid JSON, fetching from a failed network request). You can handle these situations without crashing your application using a `try...catch` block.

try {
    // Code that might throw an error
    let riskyFunction = () => { throw new Error("Something went wrong!"); };
    let result = riskyFunction();
} catch (error) {
    // Code to run if an error occurs
    console.error(`An error happened: ${error.message}`);
} finally {
    // Code that runs regardless of an error
    console.log('Operation finished.');
}

Common JavaScript Error Types

SyntaxError

Why it occurs: The code violates the rules of JavaScript's grammar. This is like a typo or grammatical mistake in a sentence. It's caught by the engine *before* any code is run.

Example

let x = 5 ;) 
 // Unmatched closing parenthesis
if (x > 3) { 
 // Missing closing curly brace
console.log("Hello");

How to Solve It:

  1. Look for typos, missing parentheses `()`, brackets `[]`, or curly braces `{}`.
  2. Check for missing commas in objects or arrays.
  3. Ensure keywords like `function`, `if`, `for` are spelled correctly.
  4. Use a code editor with syntax highlighting to spot these easily.

ReferenceError

Why it occurs: You tried to use a variable or function that hasn't been declared or is not accessible in the current scope.

Example

console.log(myVariable); 
 // myVariable was never declared

function myFunction() {
    let localScopeVar = "I'm local";
}
console.log(localScopeVar); 
 // Can't access from outside

How to Solve It:

  1. Check for typos in the variable or function name. Case matters! `myVar` is not `myvar`.
  2. Make sure you declared the variable with `let`, `const`, or `var` before using it.
  3. Verify that the variable is in the correct scope. If it's inside a function, you can't use it outside that function.

TypeError

Why it occurs: You performed an operation on a value of the wrong data type. This is the most common error in JavaScript.

Example

let x = null;
console.log(x.toUpperCase()); 
 // TypeError: Cannot read properties of null

let num = 123;
num.push(4); 
 // TypeError: num.push is not a function

How to Solve It:

  1. Use `console.log(typeof myVariable)` to check the data type of your variable right before the error occurs.
  2. If you get "Cannot read properties of undefined/null", it means the variable you're trying to access a property on is `undefined` or `null`. Find out why.
  3. Ensure you are calling methods on the correct object type (e.g., `.push()` on an array, `.toUpperCase()` on a string).

RangeError

Why it occurs: You used a number that is outside the allowed range for a specific operation.

Example

let arr = new Array(-1); 
 // RangeError: Invalid array length

let num = 5.123;
num.toFixed(101); 
 // RangeError: toFixed()
 digits argument must be between 0 and 100

How to Solve It:

  1. Read the documentation for the function you're using to understand its valid range of inputs.
  2. Add checks to ensure your numbers are within the valid bounds before passing them to the function.

Logical Errors

Why it occurs: The code is syntactically perfect and runs without crashing, but it doesn't produce the expected result. The computer is doing exactly what you told it to do, but what you told it to do was wrong.

Example

// Goal: calculate total price with tax
let price = 100;
let tax = 0.08;
let total = price + tax; 
 // Logic Error! total is 100.08, not 108 

// Correct logic: let total = price * (1 + tax);

How to Solve It:

  1. This is the hardest type to solve. The console won't help you.
  2. Go back to your plan. What did you expect each step to do?
  3. Use `console.log()` at every step to see the intermediate values of your variables.
  4. Step through your code with a debugger to watch the logic execute line by line.
  5. Write automated tests that check if your function's output matches the expected output for various inputs.
