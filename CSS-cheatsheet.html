üìñ CSS Fundamentals

What is CSS?

CSS (Cascading Style Sheets) is the language for styling web pages. It controls the colors, fonts, layout, and overall visual appearance of HTML elements.

Linking CSS

For organization and performance, always use an **External Stylesheet**. Link it in your HTML's ``. This is the professional standard.

rel="stylesheet" href="styles.css">

The Cascade, Specificity, and Inheritance

This is the most important concept to understand. It's what the "C" in CSS stands for and it determines which style rule is applied when multiple rules conflict.

Cascade

The cascade is the process of combining different stylesheets and resolving conflicts. The browser considers rules in this order:
1. **Browser default styles** (e.g., blue links)
2. **User agent styles** (user's custom CSS)
3. **Author styles** (your CSS file)
4. **Inline styles** (e.g., `style="..."`)
5. `!important` rules (use with caution!)

Specificity

When multiple rules target the same element, the most specific one wins. Specificity is calculated like a score:

  • IDs (`#id`): Most specific
  • Classes (`.class`), attributes (`[type="..."]`), pseudo-classes (`:hover`): Medium specificity
  • Elements (`p`), pseudo-elements (`::before`): Least specific

Inheritance

Some CSS properties, like `color` and `font-family`, are passed down (inherited) from parent elements to their children. Others, like `border` and `padding`, are not. You can force inheritance with the `inherit` keyword.

üéØ Selectors

Selectors are patterns that target HTML elements. Precision is key. A good developer knows the right selector for the job.

Basic Selectors

  • Element: `p`
  • Class: `.my-class`
  • ID: `#unique-id`
  • Universal: `*`
  • Grouping: `h1, h2, h3`

Combinators

  • Descendant: `article p`
  • Direct Child: `ul > li`
  • Adjacent Sibling: `h1 + p`
  • General Sibling: `h1 ~ p`

Pseudo-Classes

Select elements based on their state or position.

  • `:hover`, `:focus`, `:active`
  • `:first-child`, `:last-child`
  • `:nth-child(2n)` (even items)
  • `:not(.special)` (not having a class)

Pseudo-Elements & Attributes

  • Pseudo-Elements: `::before`, `::after`, `::first-letter`, `::selection`
  • Attribute: `a[target="_blank"]`
  • Attribute Contains: `a[href*="google"]`

üëª Pseudo-classes & Pseudo-elements

These powerful selectors allow you to style elements based on their state, position, or even create "virtual" elements that don't exist in the HTML, allowing for incredible creative control without cluttering your markup.

Pseudo-classes (The 'State' Selectors)

Pseudo-classes select an element when it's in a specific state. The selector starts with a single colon (:).

User Action States

React to what the user is doing. These are essential for any interactive element.

  • :hover - When the user's mouse is over the element.
  • :focus - When the element (like an input field) is focused.
  • :active - While the element (like a button) is being clicked.

Positional & Structural

Style elements based on their position in the document tree. Perfect for lists and tables.

  • :first-child / :last-child - Selects the first or last item in a group of siblings.
  • :nth-child(n) - Selects an item by its position. (2n) for even, (2n+1) for odd, or just (3) for the third item.
  • :only-child - Selects an element if it's the only child of its parent.

Form States

Provide instant visual feedback for form validation and interaction.

  • :checked - For checked checkboxes or radio buttons.
  • :disabled / :enabled - For interactive elements that are disabled or enabled.
  • :required / :optional - For required/optional form fields.
  • :valid / :invalid - For fields whose content is valid/invalid based on type or pattern attributes.

Logical Negation

The :not() pseudo-class lets you select all elements that do not match a certain selector. Incredibly useful for exceptions.

/* Style all buttons except the primary one */
button:not(.primary-action) {
  background-color: grey;
}

Pseudo-elements (The 'Virtual' Selectors)

Pseudo-elements let you style a specific part of a selected element. They act like adding an extra element without touching the HTML. The selector starts with a double colon (::).

::before & ::after (The Workhorses)

These create a virtual first child and last child inside the selected element. They must have a content property to work. They are used for tooltips, custom list bullets, decorative shapes, and more.

/* Add quotation marks around a blockquote */
blockquote::before {
  content: '"';
  font-size: 2em;
}
blockquote::after {
  content: '"';
  font-size: 2em;
}

Typographic Control

Style specific parts of the text content within an element.

  • ::first-letter - Style the very first letter (e.g., for a "drop cap" effect).
  • ::first-line - Style the first line of text in a block-level element.
  • ::selection - Style the part of the document that has been highlighted by the user.
  • ::placeholder - Style the placeholder text in an <input> or <textarea>.

üé® Colors & Units

Choosing the right units and color formats is fundamental for creating scalable and maintainable designs.

Color Formats

  • Hex: `#007991` (most common)
  • RGB: `rgb(0, 121, 145)`
  • RGBA: `rgba(0, 121, 145, 0.8)` (includes alpha/transparency)
  • HSL: `hsl(191, 100%, 28%)` (Hue, Saturation, Lightness - very intuitive)
  • Keywords: `red`, `blue`, `transparent`

Absolute Units

Fixed values. Best for things you want to remain a constant size.

  • `px`: Pixels. The most common absolute unit.

Relative Units (‚úÖ Modern Practice)

Scale relative to another value. Essential for responsive design.

  • `rem`: Relative to the root `` font size. **Excellent for scalability and accessibility.**
  • `em`: Relative to the parent element's font size.
  • `%`: Relative to the parent element's size.
  • `vw`, `vh`: 1% of the viewport's width or height.

üìù Typography

Styling text is one of the most important parts of web design. Good typography makes your site readable and beautiful.

Font Family

Always provide fallback fonts in case the user's system doesn't have the primary one.

body {
  font-family: "Poppins", "Helvetica Neue", sans-serif;
}

Size & Weight

Use `rem` for `font-size` to respect user's browser settings. `font-weight` can be a number (100-900) or keyword.

h1 {
  font-size: 2.5rem;
  font-weight: bold; /* or 700 */
}

Line Height & Spacing

Use a unitless value for `line-height` for better scaling. `letter-spacing` and `word-spacing` control micro-typography.

p {
  line-height: 1.6; /* 1.6 times the font size */
  letter-spacing: 0.5px;
}

Text Alignment & Decoration

Control the horizontal alignment and add decorative lines.

.title { text-align: center; }
a { text-decoration: none; }
.uppercase { text-transform: uppercase; }

üìè CSS Units: px, em, rem, %, vw, vh, and More

CSS offers a variety of units for sizing and spacing. Choosing the right one is key to building responsive, accessible layouts.

Unit Description Best Use Cases
px Pixels. Fixed, absolute size. Not affected by parent or root font size. Borders, fine details, when exact control is needed.
em Relative to the font-size of the element. 2em = 2 √ó current font size. Padding, margin, spacing that should scale with text size.
rem Relative to the root element's font-size (usually <html>). Font sizes, consistent spacing across components.
% Percentage of the parent element's size (width, sometimes height). Fluid layouts, images, containers that scale with parent.
vw, vh Viewport width/height. 1vw = 1% of viewport width. Full-screen sections, hero banners, responsive typography.
fr Fractional unit for CSS Grid. Divides available space. Grid layouts (columns/rows).
ch, ex Width of "0" (ch) or x-height (ex) of current font. Monospaced layouts, input widths.

Sizing Box

Summary: When to Use Each Unit

  • px: For borders, icons, and when you need exact, fixed sizing.
  • em: For spacing that should scale with the element's font size (e.g., button padding).
  • rem: For consistent font sizes and spacing across your site, respecting user preferences.
  • %: For fluid layouts that scale with their parent container.
  • vw/vh: For full-screen sections, hero images, or responsive typography that adapts to the viewport.
  • fr: For dividing space in CSS Grid layouts.

Tip: Use rem for most font sizes and layout spacing for accessibility and consistency. Use em for component-internal spacing. Use px only when you need pixel-perfect control.

üìå Positioning

The `position` property specifies how an element is positioned. It can remove elements from the normal document flow to create overlays, sticky headers, and more.

`static` (default)

The element exists in the normal document flow.

`relative`

The element's box is offset from its original position. The space it would have occupied is preserved.

`absolute`

The element is removed from the flow and positioned relative to its nearest *positioned* ancestor. Essential for tooltips and modals.

`fixed`

The element is positioned relative to the browser window. It won't move when the page is scrolled. Perfect for a static header or footer.

`sticky`

A hybrid of `relative` and `fixed`. The element scrolls normally until it hits a specified offset (e.g., `top: 0`), at which point it "sticks" in place. This search bar is an example!

Transitions

Transitions provide a way to smoothly animate changes to CSS properties over a given duration. They are perfect for interactive feedback, like on hover.

.my-button {
  background-color: #007991;
  transform: scale(1);
  /* property | duration | timing-function | delay */
  transition: background-color 0.3s ease-out, transform 0.3s ease-out;
}

.my-button:hover {
  background-color: #004d40;
  transform: scale(1.1); /* Grows slightly on hover */
}

Go beyond the basics with modern CSS features that allow for powerful theming and stunning visual effects.

CSS Variables

Also known as Custom Properties. Define a value once and reuse it everywhere. Incredible for theming and clean code. This entire page uses them for its light/dark modes!

:root {
  --primary-color: #007991;
}
.button {
  background-color: var(--primary-color);
}

Shadows: `box-shadow` & `text-shadow`

Add depth to your elements. You can create multiple layers by comma-separating the values.

.card {
  /* offset-x | offset-y | blur | spread | color */
  box-shadow: 0 10px 20px rgba(0,0,0,0.1);
}
.title {
  text-shadow: 2px 2px 4px rgba(0,0,0,0.2);
}

Gradients

Create smooth transitions between two or more colors. The headers on this page use them.

.hero {
  background: linear-gradient(to right, #1f4037, #99f2c8);
}
.circle {
  background: radial-gradient(circle, #fff, #ddd);
}

Blend Modes

Control how an element's content should blend with the content of the element's parent and background.

.image-overlay {
  background-color: var(--primary-color);
  /* multiplies the darks */
  mix-blend-mode: multiply;
}

Mobile-First Media Query Syntax

Start with base styles for mobile, then use `min-width` to add or override styles for larger screens. This is more efficient and leads to cleaner code.

/* --- Default styles (Mobile) --- */
.grid-container {
  display: grid;
  grid-template-columns: 1fr; /* A single column on mobile */
  gap: 10px;
}

/* --- Tablet Styles (and up) --- */
@media (min-width: 768px) {
  .grid-container {
    grid-template-columns: 1fr 1fr; /* Two columns on tablets */
  }
}

/* --- Desktop Styles (and up) --- */
@media (min-width: 1200px) {
  .grid-container {
    grid-template-columns: repeat(3, 1fr); /* Three columns on desktop */
  }
}
