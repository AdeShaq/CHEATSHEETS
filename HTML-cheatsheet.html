📖 Introduction to HTML

What is HTML?

HTML stands for HyperText Markup Language. It is the standard language used to create and design the structure of web pages. Think of it as the skeleton of a website; it provides the fundamental framework and content, which is then styled by CSS and made interactive by JavaScript.

Basic Document Structure

Every HTML document follows a standard structure. This boilerplate code is the starting point for any webpage.

<!DOCTYPE html>
<html lang="en">
    <head>
        <!-- Metadata about the page goes here (title, styles, scripts) -->
        <meta charset="UTF-8">
        <meta name="viewport" content="width=device-width, initial-scale=1.0">
        <title>My Web Page</title>
    </head>
    <body>
        <!-- All visible content of the page goes here -->
        <h1>Hello, World!</h1>
        <p>This is my first web page.</p>
    </body>
</html>

Anatomy of an HTML Element

An HTML element is an individual component of an HTML document. Most elements consist of a start tag, content, and an end tag.

  • Opening Tag: Consists of the element name (e.g., `p`) enclosed in angle brackets. This marks the beginning of the element.
  • Content: The text or other HTML elements that are inside the element.
  • Closing Tag: The same as the opening tag, but with a forward slash before the element name. This marks the end of the element.
  • Empty Elements: Some elements, like `<br>` or `<img>`, don't have a closing tag because they don't enclose content. These are called empty or void elements.

Attributes

Attributes provide additional information about an element and are always specified in the opening tag. They come in name/value pairs like `name="value"`.

<!-- The 'href' attribute specifies the URL for the link -->
<a href="https://www.example.com">Visit Example.com</a>

<!-- The 'src' and 'alt' attributes for an image -->
<img src="image.jpg" alt="A descriptive text for the image">

<!-- The 'class' attribute is used for styling with CSS -->
<p class="highlight-text">This paragraph has a class.</p>

🧠 Document Head & Metadata

The <head> element is a container for metadata (data about data) and is not displayed on the page itself. It contains information that is essential for the browser to render the page correctly and for search engines to understand the page's content.

Linking CSS and JavaScript

The most important function of the head is to link external resources.

<head>
    <!-- Defines the character encoding for the document. UTF-8 is standard. -->
    <meta charset="UTF-8">
    
    <!-- Makes the website responsive on different devices. Crucial for mobile-first design. -->
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    
    <!-- Sets the title of the browser tab and is important for SEO. -->
    <title>HTML Cheatsheet - Complete Reference</title>
    
    <!-- ✅ BEST PRACTICE: Links to an external CSS stylesheet for styling. -->
    <link rel="stylesheet" href="styles.css">

    <!-- Example of linking a library like Tailwind CSS via CDN -->
    <script src="https://cdn.tailwindcss.com"></script>

    <!-- Links to a favicon (the icon in the browser tab). -->
    <link rel="icon" href="favicon.ico" type="image/x-icon">
    
    <!-- SEO-related metadata: description and keywords. -->
    <meta name="description" content="A comprehensive guide to HTML.">
    <meta name="keywords" content="HTML, Cheatsheet, Web Development">
    
    <!-- ✅ BEST PRACTICE: Includes JavaScript with 'defer' to prevent render-blocking. -->
    <script src="app.js" defer></script>
</head>

📝 Text Formatting & Semantics

HTML provides a wide range of tags to structure and give meaning to text content. Using the correct semantic tag improves accessibility and SEO.

Headings and Paragraphs

<h1>Main Heading - Only one per page</h1>
<h2>Subheading Level 2</h2>
<h3>Subheading Level 3</h3>
<!-- ...down to h6 -->

<p>This is a paragraph of text. Paragraphs are block-level elements, so they start on a new line.</p>

<p>Use <br> for a line break within a paragraph.
This text appears on a new line.</p>

<hr> <!-- This creates a thematic break or horizontal rule -->

Common Semantic Text Tags

Emphasis and Importance

  • <strong>: Indicates strong importance, seriousness, or urgency. Browsers typically render this as bold.
  • <em>: Indicates stress emphasis. Browsers typically render this as italic.
<p>You <strong>must</strong> complete this task by Friday.</p>
<p>I <em>really</em> like this design.</p>

Quotes and Citations

  • <blockquote>: For long, multi-line quotations that are indented.
  • <q>: For short, inline quotations that are enclosed in quotes.
  • <cite>: For citing the title of a creative work.
<blockquote cite="source_url">
  <p>A long quote goes here.</p>
</blockquote>
<p>He said, <q>Hello, world!</q>.</p>
<p>My favorite book is <cite>The Hobbit</cite>.</p>

Edits and Annotations

  • <mark>: Highlights text for reference, like a highlighter pen.
  • <ins>: Represents inserted (added) text.
  • <del>: Represents deleted (removed) text.
  • <s>: Represents text that is no longer accurate or relevant (strikethrough).
<p>The important part is <mark>this</mark>.</p>
<p>Price: <del>$100</del> <ins>$50</ins>!</p>

Code and Technical Text

  • <code>: For a short fragment of computer code.
  • <pre>: For a larger block of preformatted text (preserves whitespace and line breaks).
  • <kbd>: Represents keyboard input.
<p>Use the <code>console.log()</code> function.</p>
<pre><code>
function greet() {
  return "Hello";
}
</code></pre>
<p>Press <kbd>Ctrl</kbd> + <kbd>C</kbd> to copy.</p>

🔗 Links & Navigation

Links, created with the anchor tag <a>, are the foundation of the web, allowing users to navigate between pages and resources.

Types of Links

External Link

Links to a different website. Always use the full URL (including `https://`).

<a href="https://www.google.com">Go to Google</a>

Internal Link (Relative)

Links to another page within the same website.

<a href="/about.html">About Us</a>
<a href="contact/form.html">Contact</a>

Page Anchor Link

Links to a specific section on the same page. The `href` matches the `id` of the target element.

<!-- Link -->
<a href="#section-2">Jump to Section 2</a>

<!-- ... some content ... -->

<!-- Target Element -->
<h2 id="section-2">This is Section 2</h2>

Email and Phone Links

Opens the user's default email client or phone dialer.

<a href="mailto:contact@example.com">Email Us</a>
<a href="tel:+1234567890">Call Us</a>

The `target` Attribute

The target attribute specifies where to open the linked document.

⚠️ Security Note

When using target="_blank", always add rel="noopener noreferrer" to prevent potential security vulnerabilities (phishing attacks).

<!-- Opens in a new tab or window (standard practice) -->
<a href="..." target="_blank" rel="noopener noreferrer">New Tab</a>

<!-- Opens in the same frame (default behavior) -->
<a href="..." target="_self">Same Frame</a>

The `<nav>` Element

The <nav> semantic tag is used to wrap major navigation blocks, making it clear to screen readers and search engines what the primary navigation links for the site are.

<nav>
  <ul>
    <li><a href="/">Home</a></li>
    <li><a href="/products">Products</a></li>
    <li><a href="/about">About Us</a></li>
  </ul>
</nav>

📋 Lists

HTML provides three different types of lists for organizing content.

Unordered Lists (`<ul>`)

Used for a collection of items where the order does not matter. Typically rendered with bullet points.

<ul>
  <li>Milk</li>
  <li>Bread</li>
  <li>Cheese</li>
</ul>

Ordered Lists (`<ol>`)

Used for a collection of items where the order is important. Typically rendered with numbers.

<h4>Standard Numbered List</h4>
<ol>
  <li>First, open the application.</li>
  <li>Second, enter your details.</li>
  <li>Third, click submit.</li>
</ol>

<h4>List with Roman Numerals, starting from 3</h4>
<ol type="I" start="3">
  <li>Third item</li>
  <li>Fourth item</li>
</ol>

Description Lists (`<dl>`)

Used for a list of terms and their corresponding descriptions. Perfect for glossaries or key-value pairs.

<dl>
  <dt>HTML</dt>
  <dd>HyperText Markup Language, the standard language for creating web pages.</dd>

  <dt>CSS</dt>
  <dd>Cascading Style Sheets, used for styling the presentation of web pages.</dd>
</dl>

🖼️ Images & Multimedia

HTML allows you to embed images, audio, video, and other documents directly into your pages.

Images (`<img>`)

The <img> tag is an empty element used to embed an image. The alt attribute is mandatory for accessibility.

⚠️ Accessibility is Key!

The alt attribute provides alternative text for an image if it cannot be displayed. It's read aloud by screen readers, making your site accessible to visually impaired users. Never skip the `alt` attribute. If the image is purely decorative, use an empty `alt=""`.

<img 
  src="images/logo.png" 
  alt="The company logo for XenoDesigns" 
  width="200" 
  height="50"
>

Figures and Captions

Use <figure> to encapsulate self-contained content (like an image, diagram, or code block) and <figcaption> to provide a caption.

<figure>
  <img src="chart.png" alt="A bar chart showing monthly sales.">
  <figcaption>Fig.1 - Monthly Sales Growth, Q3 2025.</figcaption>
</figure>

Audio and Video

HTML5 introduced native support for audio and video without requiring plugins.

Audio (`<audio>`)

Embeds sound content. The `controls` attribute provides play/pause/volume controls.

<audio controls>
  <source src="sound.mp3" type="audio/mpeg">
  <source src="sound.ogg" type="audio/ogg">
  Your browser does not support the audio element.
</audio>

Video (`<video>`)

Embeds video content. The `poster` attribute shows an image before the video plays.

<video controls width="400" poster="thumbnail.jpg">
  <source src="movie.mp4" type="video/mp4">
  <source src="movie.webm" type="video/webm">
  <track kind="subtitles" src="subtitles_en.vtt" srclang="en" label="English">
  Your browser does not support the video tag.
</video>

📊 Tables

HTML tables are used to display tabular data in rows and columns. While once used for page layout, they should now only be used for their intended purpose: presenting data.

Advanced Table Structure

For complex tables, use semantic elements to define the header, body, and footer sections. This improves accessibility and allows for better styling and browser rendering. The `scope` attribute explicitly associates header cells with the data they describe.

<table>
  <caption>Employee Contact Information</caption>
  <thead>
    <tr>
      <th scope="col">Name</th> <!-- 'col' scope for column header -->
      <th scope="col">Email</th>
      <th scope="col">Phone</th>
    </tr>
  </thead>
  <tbody>
    <tr>
      <th scope="row">John Doe</th> <!-- 'row' scope for row header -->
      <td>john.doe@example.com</td>
      <td>555-1234</td>
    </tr>
    <tr>
      <th scope="row">Jane Smith</th>
      <td>jane.smith@example.com</td>
      <td>555-5678</td>
    </tr>
  </tbody>
  <tfoot>
    <tr>
      <td colspan="3">Total Employees: 2</td> <!-- colspan merges cells horizontally -->
    </tr>
  </tfoot>
</table>

⌨️ Forms & User Input

Forms are a cornerstone of interactive web applications, allowing users to submit data to a server.

The `<form>` Element

The <form> element is a container for all form controls.

<form action="/submit-data" method="POST" enctype="multipart/form-data">
  <!-- Form controls go here... -->
</form>
  • action: The URL where the form data will be sent.
  • method: The HTTP method to use. GET appends data to the URL (for search forms), while POST sends it in the request body (for sensitive or large data).
  • enctype: Specifies how the form-data should be encoded. Use multipart/form-data when your form includes file uploads (`<input type="file">`).

Common Input Types

Text and Password

<label for="username">Username:</label>
<input type="text" id="username" name="username" required>

<label for="password">Password:</label>
<input type="password" id="password" name="password">

Checkboxes and Radio Buttons

<!-- Checkboxes allow multiple selections -->
<input type="checkbox" id="terms" name="terms">
<label for="terms">I agree to the terms.</label>

<!-- Radio buttons allow only one selection in a group -->
<input type="radio" id="male" name="gender" value="male">
<label for="male">Male</label>
<input type="radio" id="female" name="gender" value="female">
<label for="female">Female</label>

Dropdowns and Text Areas

<label for="country">Country:</label>
<select id="country" name="country">
  <option value="us">United States</option>
  <option value="ca">Canada</option>
</select>

<label for="comment">Comment:</label>
<textarea id="comment" name="comment" rows="4"></textarea>

Buttons and File Uploads

<button type="submit">Submit Form</button>
<input type="reset" value="Clear Form">

<label for="avatar">Choose a profile picture:</label>
<input type="file" id="avatar" name="avatar"
       accept="image/png, image/jpeg">

🔠 HTML Entities

An HTML entity is a piece of text ("string") that begins with an ampersand (`&`) and ends with a semicolon (`;`). They are used to display reserved characters (which would otherwise be interpreted as HTML code) and characters that are not on a standard keyboard.

Why are entities necessary?

If you wanted to display the text `<p>` on your webpage literally, you couldn't just type it. The browser would interpret it as the start of a new paragraph element. By using an entity, `&lt;p&gt;`, you tell the browser to display the characters `<p>` instead of processing them as code.

Commonly Used HTML Entities

Result Description Entity Name Entity Number
< Less-than sign &lt; &#60;
> Greater-than sign &gt; &#62;
& Ampersand &amp; &#38;
" Double quotation mark &quot; &#34;
' Single quotation mark (apostrophe) &apos; &#39;
Non-breaking space &nbsp; &#160;
© Copyright symbol &copy; &#169;
® Registered trademark &reg; &#174;
€ Euro sign &euro; &#8364;

Usage Example

<!-- Correctly displaying HTML code as text -->
<p>To create a paragraph, use the <code>&lt;p&gt;</code> tag.</p>

<!-- Using copyright and non-breaking space -->
<p>Copyright &copy;&nbsp;2025 XenoDesigns</p>

🖼️ Iframes

An `<iframe>` (inline frame) is used to embed another HTML document within the current one. This is commonly used for embedding content from other sites, like YouTube videos, Google Maps, or advertisements.

Security and Performance Considerations

Iframes can pose security risks and slow down your page. They create a separate browsing context. Use the `sandbox` attribute to restrict the iframe's capabilities and the `loading="lazy"` attribute to defer loading until it's near the viewport.

Embedding a Web Page

<iframe 
  src="https://www.example.com" 
  width="600" 
  height="400"
  title="An embedded version of Example.com"
  loading="lazy"
  sandbox="allow-scripts allow-same-origin">
</iframe>

Embedding a YouTube Video

Most video platforms provide a pre-made `<iframe>` snippet for easy embedding.

<iframe 
  width="560" 
  height="315" 
  src="https://www.youtube.com/embed/dQw4w9WgXcQ" 
  title="YouTube video player" 
  frameborder="0" 
  allow="accelerometer; autoplay; clipboard-write; encrypted-media; gyroscope; picture-in-picture" 
  allowfullscreen>
</iframe>

🏛️ Semantic HTML Layout

Semantic HTML elements clearly describe their meaning to both the browser and the developer. Using them improves SEO, accessibility, and code readability.

Why Not Just Use `<div>` for Everything?

While you can build a whole site with <div> tags, it creates "div-itis" and provides no context. A screen reader doesn't know if a <div> is a header, a footer, or a main article. Semantic tags solve this problem.

Common Layout Elements

<body>
  <!-- The header for the page, often contains logo, nav, search -->
  <header>
    <h1>My Awesome Website</h1>
    <nav>
      <!-- Main navigation links -->
    </nav>
  </header>

  <!-- The main, unique content of the page. Only one per page. -->
  <main>
    <!-- A self-contained piece of content (e.g., blog post, news story) -->
    <article>
      <h2>Article Title</h2>
      <!-- A thematic grouping of content, typically with a heading -->
      <section>
        <h3>Introduction</h3>
        <p>...</p>
      </section>
    </article>
    
    <!-- Content indirectly related to the main content (e.g., sidebar, ads) -->
    <aside>
      <h3>Related Links</h3>
    </aside>
  </main>

  <!-- The footer for the page, often contains copyright, contact info, links -->
  <footer>
    <p>&copy; 2025 XenoDesigns. All rights reserved.</p>
  </footer>
</body>

🌐 Global Attributes

Global attributes are attributes that can be used on all HTML elements.

Identification: `id` and `class`

  • id: A unique identifier for an element on the page. Used for page anchors and selecting a specific element with CSS/JS.
  • class: One or more identifiers to group elements. Used for styling multiple elements with CSS or selecting them with JS.

Styling and Content: `style` and `title`

  • style: Applies inline CSS styles directly to an element. (Use external stylesheets for better practice).
  • title: Provides extra information about an element, often shown as a tooltip on hover.

Language and Direction: `lang` and `dir`

  • lang: Specifies the language of the element's content.
  • dir: Specifies the text direction (e.g., `ltr` or `rtl`).

Custom Data: `data-*`

  • data-*: A way to store custom private data on an element, which can be easily accessed by JavaScript (`element.dataset`).
<!-- 'id' must be unique, 'class' can be shared -->
<div id="main-content" class="container theme-dark">...</div>

<!-- Inline style and a tooltip title -->
<button style="color: red;" title="Click to delete item">Delete</button>

<!-- Custom data attribute for JavaScript use -->
<div data-user-id="123" data-role="admin">User Profile</div>

<!-- Toggling visibility -->
<p hidden>This paragraph is not visible.</p>

<!-- Controlling keyboard navigation order -->
<input tabindex="2">
<input tabindex="1">

♿ Accessibility (ARIA)

Writing accessible HTML is not just a best practice; it's a necessity for creating inclusive web experiences. WAI-ARIA (Web Accessibility Initiative – Accessible Rich Internet Applications) provides attributes to make web content more accessible to people with disabilities, especially when using assistive technologies like screen readers.

The First Rule of ARIA

Don't use ARIA if you can use a native semantic HTML element. For example, instead of creating a `<div>` and adding `role="button"`, just use a `<button>` element. ARIA is for bridging gaps where native semantics are not sufficient.

Key ARIA Concepts: Roles, States, and Properties

Roles

Roles define what an element is or does. They tell assistive technologies the purpose of an element when it's not a standard semantic tag.

<!-- This div is now identified as a button -->
<div role="button" tabindex="0">Click Me</div>

<!-- This list is identified as a navigation menu -->
<ul role="menubar">
  <li role="menuitem">Home</li>
</ul>

Properties

Properties give elements extra meaning or context. `aria-label` is crucial for providing accessible names to elements that have no visible text (like an icon button).

<!-- Provides a label for a button with only an icon -->
<button aria-label="Close">
  <span aria-hidden="true">X</span>
</button>

<!-- Connects an input to its description -->
<input id="pwd" aria-describedby="pwd-help">
<div id="pwd-help">Must be 8+ characters.</div>

States

States represent the current condition of an element, which can change based on user interaction. They are usually managed with JavaScript.

<!-- An expandable menu button -->
<button aria-expanded="false">Menu</button>

<!-- A required form field -->
<input type="text" aria-required="true">

<!-- A selected tab in a tab list -->
<li role="tab" aria-selected="true">Profile</li>

🎨 Graphics: SVG & Canvas

HTML provides two powerful ways to create graphics directly in the browser: SVG for scalable vector images and Canvas for pixel-based drawing.

SVG (Scalable Vector Graphics)

SVG is an XML-based language for describing two-dimensional vector graphics. Since it's vector-based, it scales to any size without losing quality, making it perfect for logos, icons, and interactive charts.

SVG Advantages

  • Scalable: Looks crisp at any resolution.
  • Accessible: Text within SVG is searchable and selectable.
  • Styleable with CSS: You can style SVG elements with CSS.
  • Interactive with JS: You can add event listeners to SVG elements.
  • Small File Sizes: Often smaller than raster images for simple graphics.

Inline SVG Example

You can embed SVG directly in your HTML document.

<svg width="100" height="100" aria-labelledby="svgTitle">
  <title id="svgTitle">A red circle</title>
  <circle cx="50" cy="50" r="40" stroke="black" stroke-width="3" fill="red" />
</svg>

Canvas

The <canvas> element provides a blank drawing surface. You use JavaScript to draw graphics on it pixel by pixel. It is ideal for dynamic applications like games, data visualizations, and photo editing tools.

Canvas Advantages

  • High Performance: Excellent for rendering a large number of objects and complex animations.
  • Pixel Manipulation: Allows for direct, low-level control over every pixel.
  • Ideal for Dynamic Graphics: Well-suited for applications where the scene changes frequently.

Canvas Example

The HTML is simple; all the drawing logic is in JavaScript.

<!-- HTML -->
<canvas id="myCanvas" width="200" height="100"></canvas>

<!-- JavaScript (in a <script> tag) -->
<script>
  const canvas = document.getElementById('myCanvas');
  const ctx = canvas.getContext('2d');
  
  ctx.fillStyle = 'blue';
  ctx.fillRect(10, 10, 150, 80);
</script>

✅ HTML Best Practices & Final Checklist

Writing good HTML is not just about making things appear on a screen. It's about creating documents that are robust, maintainable, and accessible to all users, including those with disabilities.

Validate Your HTML

Use tools like the W3C Markup Validation Service to check for syntax errors. Valid HTML is more predictable and less likely to have rendering issues across browsers.

Use Semantic Tags

Always choose the HTML element that best describes its content. Use <button> for buttons, <nav> for navigation, and <p> for paragraphs. This is the #1 rule for accessibility and SEO.

Provide `alt` Text for Images

Every <img> tag must have an alt attribute. Describe the image's content and function. If it's purely decorative, provide an empty alt="" so screen readers know to skip it.

Maintain a Logical Heading Structure

Don't skip heading levels (e.g., from an <h1> to an <h3>). Use headings to create a clear, hierarchical outline of your page content. Use CSS for styling, not heading tags.

Use Labels for Form Inputs

Always associate a <label> with every form control (`<input>`, `<textarea>`, `<select>`) using the `for` attribute. This allows users to click the label to focus the input and is critical for screen readers.

Minimize `div` and `span`

Before you use a generic <div> or <span>, ask yourself: "Is there a more specific semantic element I could use instead?" Use them only for styling or grouping when no other element fits.
